# Спецификация KPI отдела закупок, снабжения и логистики

**Назначение документа:** единый источник истины для системы мониторинга KPI в приложении. Все формулы, правила оценки и входные данные заданы однозначно и привязаны к реализации.

**Источник:** Пояснительная записка к системе мотивации сотрудников (PDF). Документ переписан буква за буквой в структурированный MD для разработки мониторинга.

---

## 1. Общие положения

Система мотивации сотрудников отдела закупок, снабжения и логистики основана на ключевых показателях эффективности (KPI), которые:

- отражают эффективность выполнения прямых функциональных обязанностей;
- позволяют оценить индивидуальный и командный вклад в достижение целей компании;
- соотносятся с задачей построения объективной системы оценки эффективности и структуры оплаты труда.

За основу оценки берутся перечисленные ниже показатели и их **веса** (степень влияния на общий коэффициент). Веса приведены для примера и подлежат корректировке в соответствии с реальной обстановкой или регулярной корректировке в интересах управления отделом.

## 3. KPI 1 — Соблюдение бюджета (KPIбюджет / budget)

### 3.1. Описание

Комплексный показатель: отношение **фактически затраченного бюджета закупок** к **плановому бюджету закупок** за отчётный период.

### 3.2. Методика

- **До периода:** сотрудники по направлениям готовят бюджет закупок на основании плана закупок и передают руководителю отдела. Руководитель формирует сводный бюджет закупок на предстоящий отчётный период и защищает его перед генеральным директором. Так определяется **целевой (плановый) показатель**.
- **После периода:** суммируются все расходы, направленные на выполнение плана закупок — определяется **фактический показатель**.

### 3.3. Формула расчёта

```
KPI_бюджет_% = (Фактические затраты / Плановые затраты) × 100%
```

### 3.4. Пример расчёта

- Плановые затраты: 40 000 000,00 руб  
- Фактические затраты: 42 000 000,00 руб  

```
KPI_бюджет_% = 42 000 000 / 40 000 000 × 100% = 105%
```

Плановый бюджет превышен на 5%.

### 3.5. Правило перевода в коэффициент (0–1)

- Диапазон: 0 — критический провал, 1 — наилучший результат.
- **Правило:** от 1 отнимаем долю превышения плана (в долях единицы).

```
коэффициент = 1 − (KPI_бюджет_% − 100%)   при KPI_бюджет_% > 100%
коэффициент = 1                            при KPI_бюджет_% ≤ 100%
```

В примере: превышение 5% → коэффициент = 1 − 0,05 = **0,950**.

### 3.6. Входные данные для мониторинга

| Поле | Описание | Источник (1С / приложение) |
|------|----------|----------------------------|
| Плановые затраты | Бюджет закупок на период, руб | План/бюджет (ручной ввод или 1С) |
| Фактические затраты | Сумма расходов по закупкам за период, руб | Заказы поставщикам / оплаты за период |

## 4. KPI 2 — Доля срочных (внеплановых) закупок (KPIвнеплан / urgent)

### 4.1. Описание

Относительная величина: доля срочных закупок в общем объёме закупок за отчётный период. Выражается отношением **количества срочных (внеплановых) закупок** к **общему количеству закупок**.

Цель — контроль и повышение качества планирования закупок.

**В расчёт не включаются** закупки по заявкам, когда ТМЦ объективно не входят в план закупок (оборудование, инструменты, расходные материалы для рекламаций, материалы, не предусмотренные спецификациями из-за упущений при проектировании/осмечивании и т.д.).

### 4.2. Методика

- Перед началом периода составляется **план закупок**: объёмы, количество, сроки и стоимость ТМЦ для обеспечения производства и строительства. План основан на сводном отчёте о потребности, сформированном по спецификациям по видам работ по договорам (объектам/проектам).
- **Внеплановая (срочная) закупка:** любая закупочная процедура в интересах производства и/или строительства, **не предусмотренная планом закупок**, при условии что закупаемые в её рамках ТМЦ **включены в соответствующие спецификации**.

### 4.3. Формула расчёта

```
KPI_внеплан_% = (Количество внеплановых (срочных) закупок / Общее количество закупок) × 100%
```

### 4.4. Пример расчёта

- Внеплановых закупок: 37  
- Общее количество закупок: 260  

```
KPI_внеплан_% = 37 / 260 × 100% = 14,23%
```

### 4.5. Правило перевода в коэффициент (0–1)

- **Правило:** от 1 отнимаем долю внеплановых закупок (в долях единицы).

```
коэффициент = 1 − (KPI_внеплан_% / 100%)
```

В примере: 1 − 0,1423 = **0,857**.

### 4.6. Входные данные для мониторинга

| Поле | Описание | Источник |
|------|----------|----------|
| Общее количество закупок | Число закупочных процедур (заказов поставщикам) за период | 1С: заказы поставщикам за период |
| Количество внеплановых закупок | Закупки, не входящие в план закупок, но по номенклатуре из спецификаций | Сопоставление плана закупок и заказов за период |

## 5. KPI 3 — Срок закупки (KPIзаявки / leadtime)

### 5.1. Описание

Показатель с привязкой к **нормативному значению** срока. Цель — не допускать затягивания закупочных процедур по заявкам, в рамках которых закупаются ТМЦ, закупка которых инициирована **вне плана закупок**.

### 5.2. Методика

- В расчёт берутся все закупочные процедуры (заказы поставщикам), выполненные по **заявкам вне плана закупок**.
- По каждой процедуре: **дата создания заявки** и **дата поступления закупаемых ТМЦ на склад**. Количество дней между этими датами — **срок поставки** (в днях).
- Частные случаи (ТМЦ, которые объективно нельзя закупить в нормативный срок) разбираются руководителем отдела в индивидуальном порядке.

### 5.3. Формула расчёта

```
KPI_заявки_% = (Среднее значение срока поставок / Нормативное значение срока поставок) × 100%
```

Срок поставки по одной процедуре = дата поступления на склад − дата создания заявки (дней).

### 5.4. Пример расчёта

- Средний срок поставок: 8 дней  
- Нормативное значение: 5 дней  

```
KPI_заявки_% = 8 / 5 × 100% = 160%
```

Нормативный срок превышен на 60%.

### 5.5. Правило перевода в коэффициент (0–1)

- **Правило:** от 1 отнимаем долю превышения нормы (превышение = KPI_заявки_% − 100% в долях).

```
коэффициент = 1 − (KPI_заявки_% − 100%) / 100%   при KPI_заявки_% > 100%
коэффициент = 1                                   при KPI_заявки_% ≤ 100%
```

В примере: превышение 60% → коэффициент = 1 − 0,60 = **0,400**.

### 5.6. Входные данные для мониторинга

| Поле | Описание | Источник |
|------|----------|----------|
| Дата создания заявки | По каждой внеплановой закупке | 1С: заявка / заказ поставщику |
| Дата поступления ТМЦ на склад | По каждой такой закупке | 1С: поступление на склад |
| Нормативное значение срока поставок | Дней | Справочник/настройки в приложении |

## 6. KPI 4 — Доля брака и возвратов (KPIкачество / quality)

### 6.1. Описание

Комплексный показатель: финансовая и количественная доля случаев, когда по результатам приёмки материал **не может быть оприходован на склад** из-за замечаний по качеству.

### 6.2. Методика

Для расчёта используются четыре величины:

1. Общая сумма закупок (руб).  
2. Суммарная стоимость ТМЦ, по которым есть замечания по качеству (руб).  
3. Общее количество закупок (закупочных процедур).  
4. Количество случаев (эпизодов) замечаний по качеству.

### 6.3. Формула расчёта

```
KPI_качество_% = 100% − (0,7 × (Стоимость ТМЦ с браком / Общая сумма закупок) + 0,3 × (Количество эпизодов брака / Общее количество закупок))
```

### 6.4. Пример расчёта

- Общая сумма закупок: 42 000 000,00 руб  
- Стоимость ТМЦ с браком: 300 000,00 руб  
- Общее количество закупок: 250  
- Количество эпизодов брака: 3  

```
KPI_качество_% = 100% − (0,7 × 300 000 / 42 000 000 + 0,3 × 3 / 250)
               = 100% − (0,7 × 0,0071 + 0,3 × 0,012)
               = 100% − (0,005 + 0,036)
               = 100% − 0,0086 = 99,99%
```

Доля поставок без брака 99,99%.

### 6.5. Правило перевода в коэффициент (0–1)

- **Правило:** коэффициент равен доле поставок без брака (KPI_качество_% в долях).

```
коэффициент = KPI_качество_% / 100%
```

В примере: 0,9999 ≈ **0,999**.

### 6.6. Входные данные для мониторинга

| Поле | Описание | Источник |
|------|----------|----------|
| Общая сумма закупок | Сумма закупок за период, руб | 1С: заказы/поступления |
| Стоимость ТМЦ с браком | Сумма ТМЦ с замечаниями по качеству за период, руб | 1С: брак, возвраты, рекламации |
| Общее количество закупок | Число закупочных процедур за период | 1С: заказы поставщикам |
| Количество эпизодов брака | Число случаев замечаний по качеству за период | 1С: документы брака/возвратов |

## 7. KPI 5 — Своевременность поставок (KPIсрок / ontime)

### 7.1. Описание

Количественный показатель: доля поставок, выполненных в **контрольный (плановый) срок**, в общем количестве поставок. Контрольный срок определяется на основании **плана закупок**. Цель — поддержание непрерывности обеспечения производства и строительных объектов ТМЦ.

### 7.2. Методика

- По сотруднику за период (в примере — месяц): **общее количество закупочных процедур**; отдельно — закупочные процедуры, по которым **дата поступления ТМЦ** совпадает с контрольным (плановым) сроком или опережает его.
- **Важно:** закупочные процедуры с планово переходящим сроком поставки или с переносом фактической поставки на следующий месяц из-за срыва срока учитываются в показателях **того месяца, в котором фактически осуществлена поставка**.

### 7.3. Формула расчёта

```
KPI_срок_% = (Количество своевременных поставок / Общее количество поставок) × 100%
```

Своевременная поставка: дата поступления ТМЦ ≤ плановый (контрольный) срок.

### 7.4. Пример расчёта

- Своевременных поставок: 26  
- Общее количество поставок: 30  

```
KPI_срок_% = 26 / 30 × 100% = 86,7%
```

### 7.5. Правило перевода в коэффициент (0–1)

- **Правило:** коэффициент равен доле своевременных поставок.

```
коэффициент = KPI_срок_% / 100%
```

В примере: **0,867**.

### 7.6. Входные данные для мониторинга

| Поле | Описание | Источник |
|------|----------|----------|
| Плановый (контрольный) срок поставки | По каждой закупочной процедуре | План закупок (1С / приложение) |
| Фактическая дата поступления ТМЦ | По каждой процедуре | 1С: поступление на склад |
| Учёт переходящих поставок | Отнесение к месяцу фактической поставки | Логика в приложении |

## 8. KPI 6 — Экономия на закупках (KPIэкономия / savings) — экспериментально

### 8.1. Описание

Относительный показатель: доля достижения **планового (целевого) показателя экономии** по ценам закупок.

### 8.2. Методика

- По каждой закупаемой позиции определяются: **плановый (средневзвешенный рыночный) уровень цены**, **средняя фактическая цена**, **нормативный (целевой) показатель экономии в %**.
- Не учитываются закупочные процедуры, в рамках которых приобретаются **уникальные ТМЦ**, цена которых не может быть определена из исторических данных.

### 8.3. Формула расчёта

```
Фактическая экономия_% = (Средняя рыночная цена − Средняя фактическая цена) / Средняя рыночная цена × 100%

KPI_экономия (коэффициент) = Фактическая экономия_% / Целевой показатель экономии в %
```

В документе итоговый KPI экономии трактуется как коэффициент (отношение достигнутой экономии к целевой).

### 8.4. Пример расчёта (номенклатура: доска сухая строганная)

- Средняя рыночная цена: 26 000 руб  
- Средняя фактическая цена: 25 725 руб  
- Целевой показатель экономии: 1,3%  

```
Фактическая экономия_% = (26 000 − 25 725) / 26 000 × 100% = 1,06%
KPI_экономия = 1,06% / 1,3% = 0,846
```

### 8.5. Правило перевода в коэффициент (0–1)

- Коэффициент уже получен формулой выше. Ограничение: на практике применяют ограничение сверху 1 (если экономия выше целевой — коэффициент = 1).

```
коэффициент = min(1, Фактическая экономия_% / Целевой показатель экономии_%)
```

### 8.6. Входные данные для мониторинга

| Поле | Описание | Источник |
|------|----------|----------|
| Средняя рыночная (плановая) цена | По номенклатуре/позиции | Справочник цен или расчёт из истории (1С) |
| Средняя фактическая цена | По закупкам за период | 1С: заказы/поступления |
| Целевой показатель экономии, % | Норматив по позиции или общий | Справочник/настройки |

## 9. KPI 7 — Качество работы с поставщиками (KPIпоставщики / suppliers)

### 9.1. Описание

Нормативный показатель. Оценивается с заданной периодичностью (в примере — **ежеквартально**). Смысл — учёт количества **улучшений**, достигнутых в работе с поставщиками.

### 9.2. Методика

Необходим зафиксированный **перечень улучшений**, за каждое из которых сотруднику засчитывается условный балл (в расчёте — 1 улучшение = 1 единица). Примеры улучшений:

1. Новый (альтернативный) поставщик (поиск + переговоры + договор + поставка).  
2. Переход с предоплаты на частичную предоплату или постоплату.  
3. Договорённость о поставках в отсрочку.  
4. Увеличение срока отсрочки.  
5. Увеличение суммы отсрочки.  
6. Сокращение стандартного срока поставки с фиксацией в договоре (доп. соглашении).  
7. Перевод поставщика на нашу форму договора.

Плановый показатель задаётся в количестве улучшений за период (в примере — 5 за квартал).

### 9.3. Формула расчёта

```
KPI_поставщики_% = (Фактическое количество улучшений / Плановое количество улучшений) × 100%
```

### 9.4. Пример расчёта

- Фактических улучшений: 4  
- Плановых улучшений: 5  

```
KPI_поставщики_% = 4 / 5 × 100% = 80%
```

### 9.5. Правило перевода в коэффициент (0–1)

- **Правило:** коэффициент равен доле выполнения плана по улучшениям. Ограничение сверху 1.

```
коэффициент = min(1, Фактическое количество улучшений / Плановое количество улучшений)
```

В примере: **0,800**.

### 9.6. Входные данные для мониторинга

| Поле | Описание | Источник |
|------|----------|----------|
| Плановое количество улучшений | На период (например квартал) | Настройки / планы в приложении |
| Фактическое количество улучшений | Учтённые улучшения по перечню за период | Ручной ввод или учёт в приложении (не из типового учёта 1С) |

## 10. Общий коэффициент KPI и мотивация

### 10.1. Формула общего коэффициента

```
KPI_общ = Σ (коэффициент_i × вес_i)
```

С учётом весов из раздела 2 (в примере для 7-го KPI использован вес 10%):

```
KPI_общ = KPI_бюджет×0,30 + KPI_внеплан×0,15 + KPI_заявки×0,05 + KPI_качество×0,10 + KPI_срок×0,20 + KPI_экономия×0,10 + KPI_поставщики×0,10
```

Пример из документа:

```
KPI_общ = 0,950×0,3 + 0,857×0,15 + 0,400×0,05 + 0,999×0,1 + 0,867×0,2 + 0,846×0,1 + 0,800×0,1 = 0,87145
```

### 10.2. Интерпретация KPI_общ

- **0** — критическая тотальная неэффективность.  
- **1** — максимальная эффективность.

### 10.3. Связь с материальной мотивацией

Задаётся **база (фиксированное значение)** в рублях. Размер материальной (финансовой) мотивации сотрудника за период:

```
Мотивация = База × KPI_общ
```

Пример: база 100 000,00 руб, KPI_общ = 0,87145 → мотивация = **87 145 руб**.

### 10.4. Управление весами

Веса ключевых показателей можно корректировать. Увеличение веса показателя увеличивает его влияние на размер мотивации и заинтересованность сотрудника в улучшении именно этого показателя.

## 11. Сводная таблица формул и правил оценки

| KPI | Код | Формула показателя (в %) | Правило: коэффициент 0–1 |
|-----|-----|--------------------------|----------------------------|
| Соблюдение бюджета | budget | Факт / План × 100% | 1 − превышение_% (если >100%) |
| Доля срочных закупок | urgent | Внеплан / Всего × 100% | 1 − доля_внеплан |
| Срок закупки | leadtime | Средний_срок / Норма × 100% | 1 − превышение_нормы_% |
| Доля брака и возвратов | quality | 100% − (0,7×брак_руб + 0,3×брак_кол) | доля_без_брака |
| Своевременность поставок | ontime | Своевр / Всего × 100% | доля_своевр |
| Экономия на закупках | savings | (Рынок − Факт)/Рынок × 100% и / Цель_% | min(1, факт_экономия_% / цель_%) |
| Качество работы с поставщиками | suppliers | Факт_улучш / План_улучш × 100% | min(1, факт/план) |
