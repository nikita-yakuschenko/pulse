### Этапы (даты по объекту)

- **На производстве:** производство, устройство свай, отгрузка с завода.  
- **Окна:** заказ/поставка, монтаж окон.  
- **На объекте:** кровля, электрика, сантехника, стяжка, наружка (наружная отделка), сдача.

В PDF также встречаются: номера заявок/документов (№2117133, №2125030), даты доставки, отметки «оплачено/не оплачены», «готовность», «монтаж завершён».

### Итоги
- Суммы по прорабам (например Илюшин 20 582 400, Грушуткин 17 728 900).  
- Общие итоги по территории/периоду.

## 2. Принципиальные моменты процесса (для системы)

1. **Объект** привязан к территории, прорабу, заказу (код + тип ПКД/МД + модель).  
2. **Временная ось** — месяц плана + дата запуска; этапы имеют целевые даты.  
3. **Этапы** — цепочка от производства (сваи, отгрузка) до сдачи (окна, кровля, электрика, сантехника, стяжка, наружка, сдача).  
4. **Статусы и комментарии** — свободный текст (ожидание теплого контура, оплата, выбор кровли и т.п.).  
5. **Группировка** — по территории (и при необходимости по прорабу для отчётов).

## 3. Предлагаемая структура раздела в приложении

### 3.1. Экран «График монтажа»

- **Верх:** выбор периода (месяц/год), при необходимости «дата запуска от/до».  
- **Группировка:** вкладки или фильтр по **территориям** (как в PDF).  
- **Таблица по объектам:** одна строка = один объект.

Колонки таблицы (по возможности совпадать с PDF):

- Населённый пункт  
- Номер договора  
- Тип (ПКД/МД)  
- Эскроу/ЕГРН  
- Проект (модель)  
- Прораб  
- Сумма  
- Дата запуска  
- Этапы (даты или статусы): производство → сваи → отгрузка → окна → монтаж окон → кровля → электрика → сантехника → стяжка → наружка → сдача  
- Статус/комментарий (одно поле или короткий список статусов + комментарий)

Дополнительно:
- Итоги по прорабам (сумма) внизу блока или в сайдбаре.  
- При клике по строке — карточка объекта с полными датами, документами, комментариями.

### 3.2. Данные (логическая модель)

Имеет смысл завести сущности:

- **Территория** — название (Н.Новгород/Арзамас/Кстово, Павлово, и т.д.), порядок для отображения.  
- **Объект (строительный/монтажный)** — населённый пункт, номер договора, тип (ПКД/МД), эскроу/ЕГРН, проект (модель), прораб (ссылка на пользователя или справочник), сумма, дата запуска, территория.  
- **Этапы объекта** — набор дат (или дата + статус) по фиксированному списку этапов: производство, сваи, отгрузка, окна, монтаж окон, кровля, электрика, сантехника, стяжка, наружка, сдача.  
- **План-период** — месяц/год (или связь «план на март 2026»), чтобы фильтровать объекты по плану.

Связь с существующей моделью **Plan** (type: production | installation): объект графика монтажа может ссылаться на план или на отдельную сущность «план на месяц», а этапы — храниться в той же записи объекта (JSON) или в отдельной таблице этапов.
