# Товары на складах (GetBalances)

HTTP-сервис возвращает остатки товаров на складах по регистру накопления **ТоварыНаСкладах** в виде единообразного JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetBalances`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив позиций номенклатуры с полями Код, Наименование, ЕдиницаИзмерения, ВидНоменклатуры, НоменклатурнаяГруппа, ДатаСреза, Остатки — массив {Склад, Количество}) и `errors` (массив строк ошибок). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр (full, Full, FULL).

| Параметр   | Назначение |
|-----------|------------|
| **code**  | Подстрока в коде номенклатуры (ПОДОБНО в запросе). |
| **name**  | Подстрока в наименовании номенклатуры (ПОДОБНО). |
| **warehouse** | Подстрока в наименовании склада; в ответ попадают только склады, где наименование содержит эту подстроку (пост-фильтрация по результатам остатков). |
| **date**  | Дата среза остатков. Форматы: **dd.MM.yyyy** или **yyyy-MM-dd**. **Если параметр не указан — по умолчанию используется текущая дата сеанса.** |
| **full**  | При **true** в **data** включаются все отфильтрованные позиции номенклатуры, в том числе с нулевыми остатками по всем складам. При **false** (по умолчанию) — только позиции, у которых остаток > 0 хотя бы на одном складе. |

В конфигураторе нужно задать шаблон URL, соответствующий имени функции **GetBalances**, либо переименовать функцию под существующий шаблон.
