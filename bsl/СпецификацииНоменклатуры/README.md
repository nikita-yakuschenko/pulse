# Спецификации номенклатуры (GetSpecifications)

HTTP-сервис возвращает спецификации номенклатуры (Справочник.СпецификацииНоменклатуры) с комплектующими (ТЧ ИсходныеКомплектующие) и датой поставки по этапам (ТЧ ЭтапыПоставок) в виде единообразного JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetSpecifications`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив спецификаций: Код, ДатаУтверждения, Ответственный, Наименование, ДатаПоставки, СтоимостьРабот, СтоимостьТранспорт, СтоимостьНакладные, СтоимостьАрендаСпецТехники, СтоимостьТехнологическиеПрисоединения, СтоимостьПроектныеРаботы, Материалы — сводка «N позиций» или массив комплектующих с Код, Материал, Количество, Единица, Цена, Сумма, НоменклатурнаяГруппа) и `errors` (массив строк ошибок). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр. В путях с символом «/» можно использовать замену `_slash_`.

| Параметр   | Назначение |
|------------|------------|
| **code**   | Подстрока в коде спецификации (ПОДОБНО). |
| **name**   | Подстрока в наименовании спецификации (ПОДОБНО). |
| **material** | Подстрока в коде или наименовании комплектующего (ПОДОБНО). При указании material без full возвращаются только строки, совпавшие по материалу; при full — спецификации, в которых встречается такой материал, с полным списком комплектующих. |
| **ngroup** | Подстрока в наименовании номенклатурной группы комплектующего (ПОДОБНО). |
| **year**   | Фильтр по году плановой даты поставки (ТЧ ЭтапыПоставок). Двузначный год трактуется как 20xx. |
| **month**  | Фильтр по месяцу (только вместе с **year**). |
| **full**   | При **true** (или при указании **material**) в поле **Материалы** возвращается полный массив комплектующих. Иначе — строка «N позиций». Параметр **full** требует указания хотя бы одного из: name, code, material. |

В конфигураторе нужно задать шаблон URL, соответствующий имени функции **GetSpecifications**, либо переименовать функцию под существующий шаблон.
