# Заказ на производство (GetMOrders)

HTTP-сервис возвращает список документов **Заказ на производство** (Документ.ЗаказНаПроизводство) в виде единообразного JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetMOrders`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив заказов: Номер, Дата, Договор (комментарий), Организация, Ответственный; при full=true дополнительно Ссылка, Продукция, Материалы — табличные части) и `errors` (массив строк ошибок). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр.

| Параметр    | Назначение |
|-------------|------------|
| **code**    | Подстрока в номере заказа (ПОДОБНО). |
| **contract**| Подстрока в комментарии заказа (договор) (ПОДОБНО). Для значения с символом «/» можно использовать замену `_slash_` или передать параметр через тело/ПараметрыЗапроса. |
| **from**, **to** | Период по дате документа. |
| **full**    | При **true** в каждую запись добавляются Ссылка, ТЧ Продукция и ТЧ Материалы (массивы строк с Номенклатура, Количество, ЕдиницаИзмерения). При **false** (по умолчанию) — только базовые поля. |

В конфигураторе нужно задать шаблон URL, соответствующий имени функции **GetMOrders**, либо переименовать функцию под существующий шаблон.
