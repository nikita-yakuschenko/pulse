# Поступление товаров и услуг (GetReceipts)

HTTP-сервис возвращает поступления товаров и услуг с документом-основанием (Сделка), табличными частями Товары и Услуги, вложениями.

**Функция в конфигураторе:** `GetReceipts`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив поступлений) и `errors` (массив строк ошибок). У каждого поступления: Номер, ДатаДокумента, Дата, Контрагент, Организация, Склад, Ответственный, СуммаДокумента, Комментарий, Проведен; при `full=true` — Сделка (документ-основание), ТЧ Товары, ТЧ Услуги, Вложения (Base64). Без `full` — сводки по Сделка и Вложения (Есть, Количество). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр (full, Full, FULL).

| Параметр | Тип | Описание |
|----------|-----|----------|
| **from** | строка (дата) | Начало периода по дате документа. Работает только вместе с **to**. |
| **to** | строка (дата) | Конец периода по дате документа. В паре с **from** задаёт фильтр по периоду. |
| **code** | строка | Подстрока в номере поступления (ПОДОБНО в запросе). |
| **date** | строка | Точная дата документа. Формат даты — по разбору в коде (поддержка формата с «-»). |
| **contractor** | строка | Подстрока в представлении контрагента (пост-фильтрация). |
| **org** | строка | Подстрока в представлении организации (пост-фильтрация). |
| **warehouse** | строка | Подстрока в представлении склада (пост-фильтрация). |
| **responsible** | строка | Подстрока в представлении ответственного (пост-фильтрация). |
| **summ** | число | Точное совпадение по сумме документа. |
| **approved** | булево | Фильтр по проведённости. Истина: `1`, `TRUE`, `YES`, `ДА`. |
| **full** | булево | При `true` — полные Сделка, ТЧ Товары, ТЧ Услуги, Вложения. При `false` или отсутствии — сводки по Сделка и Вложения. |

По умолчанию возвращаются все поступления, попадающие под переданные фильтры; при отсутствии периода (**from**/**to**) ограничения по дате нет.
