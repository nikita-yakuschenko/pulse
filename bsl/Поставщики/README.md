# Поставщики (GetSuppliers)

HTTP-сервис возвращает список контрагентов-поставщиков (Справочник.Контрагенты, признак Поставщик = Истина) в виде единообразного JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetSuppliers`. Имя шаблона URL должно совпадать с именем функции.

---

## Поля ответа (всегда)

| Поле | Описание |
|------|----------|
| Наименование | Краткое наименование контрагента |
| НаименованиеПолное | Полное наименование контрагента |
| ИНН | ИНН |
| КПП | КПП |
| ОГРН | ОГРН |
| КодПоОКПО | Код по ОКПО |
| ЮрФизЛицо | Юридическое / физическое лицо |
| СрокВыполненияЗаказа | Срок выполнения заказа поставщиком (дни) |
| ТакжеПокупатель | Является ли также покупателем (булево) |
| НеРезидент | Не является резидентом (булево) |
| Комментарий | Комментарий |
| Договоры | При `full=false` — сводка `{Есть, Количество}`. При `full=true` — массив договоров |

---

## Поля договора (при full=true)

| Поле | Описание |
|------|----------|
| Наименование | Наименование договора |
| Номер | Номер договора |
| Дата | Дата заключения (dd.MM.yyyy) |
| СрокДействия | Срок действия (dd.MM.yyyy) |
| Организация | Наша организация, с которой заключён договор |
| ВидДоговора | Вид договора |
| Валюта | Валюта взаиморасчётов |

---

## Параметры запроса

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**.

| Параметр | Назначение |
|----------|------------|
| **name** | Подстрока в наименовании контрагента (ПОДОБНО). Если не указан — ограничение не применяется. |
| **inn**  | Подстрока в ИНН контрагента (ПОДОБНО). Если не указан — ограничение не применяется. |
| **full** | `true` / `1` / `yes` / `да` — вернуть полный список договоров по каждому контрагенту. По умолчанию — только сводка (количество). |

В конфигураторе нужно задать шаблон URL, соответствующий имени функции **GetSuppliers**, либо переименовать функцию под существующий шаблон.
