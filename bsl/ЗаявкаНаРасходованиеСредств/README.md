# Заявка на расходование средств (GetPayments)

HTTP-сервис возвращает заявки на расходование средств с документом-основанием, связанными документами расчётов с контрагентом и вложениями.

**Функция в конфигураторе:** `GetPayments`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив заявок) и `errors` (массив строк ошибок). У каждой заявки: Номер, ДатаДокумента, ДатаРасхода, Ответственный, Получатель, Касса, СуммаДокумента, Статус, Комментарий, Проведен, КтоИзменилСтатус, ДатаСменыСтатуса; при `full=true` — СвязанныеДокументы (ДокументОснование, ДокументыРасчетовСКонтрагентом), Вложения (Base64). Без `full` — сводки по связям и вложениям (Есть, Количество). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр (full, Full, FULL).

| Параметр | Тип | Описание |
|----------|-----|----------|
| **from** | строка (дата) | Начало периода по дате расхода. Работает только вместе с **to**. |
| **to** | строка (дата) | Конец периода по дате расхода. В паре с **from** задаёт фильтр по периоду. |
| **code** | строка | Подстрока в номере заявки (ПОДОБНО в запросе). |
| **date** | строка | Точная дата расхода. Формат даты — по разбору в коде (поддержка формата с «-»). |
| **responsible** | строка | Подстрока в представлении ответственного (пост-фильтрация). |
| **recipient** | строка | Подстрока в представлении получателя (пост-фильтрация). |
| **contractor** | строка | Подстрока в представлении контрагента (пост-фильтрация). |
| **cash** | строка | Подстрока в представлении кассы/банковского счёта (пост-фильтрация). |
| **comment** | строка | Подстрока в комментарии заявки (пост-фильтрация, регистронезависимая). |
| **status** | строка | Подстрока в статусе заявки, напр. `Оплачен`, `Подготовлен`, `Отклонен` (пост-фильтрация, регистронезависимая). |
| **summ** | число | Точное совпадение по сумме документа. |
| **approved** | булево | Фильтр по проведённости. Истина: `1`, `TRUE`, `YES`, `ДА`. |
| **full** | булево | При `true` — полные СвязанныеДокументы и Вложения. При `false` или отсутствии — сводки (Есть, Количество). |

По умолчанию возвращаются все заявки, попадающие под переданные фильтры; при отсутствии периода (**from**/**to**) ограничения по дате расхода нет.
