# Заказы поставщикам (GetSOrders)

HTTP-сервис возвращает заказы поставщикам с вложениями и связанными документами: ПоступлениеТоваровУслуг, ЗакрытиеЗаказовПоставщикам, ЗаявкаНаРасходованиеСредств.

**Функция в конфигураторе:** `GetSOrders`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив заказов) и `errors` (массив строк ошибок). У каждого заказа: Номер, Дата, Контрагент, Организация, Склад, Ответственный, СуммаДокумента, Проведен, Комментарий, ДатаОплаты, ДатаПоступления; при `full=true` — ТЧ Товары, СвязанныеДокументы (полные списки), Вложения (Base64). Без `full` — сводки по связям и вложениям (Есть, Количество). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL** (сначала проверяется ПараметрыЗапроса). Регистр параметра `full` не учитывается (поддерживаются варианты full, Full, FULL).

| Параметр | Тип | Описание |
|----------|-----|----------|
| **from** | строка (дата) | Начало периода по дате документа. Работает только вместе с **to**. |
| **to** | строка (дата) | Конец периода по дате документа. В паре с **from** задаёт фильтр по периоду. |
| **code** | строка | Подстрока в номере заказа (ПОДОБНО в запросе). |
| **date** | строка | Точная дата документа. Формат даты — по разбору в коде (поддержка формата с «-»). |
| **supplier** | строка | Подстрока в представлении контрагента (пост-фильтрация). |
| **org** | строка | Подстрока в представлении организации (пост-фильтрация). |
| **warehouse** | строка | Подстрока в представлении склада (пост-фильтрация). |
| **responsible** | строка | Подстрока в представлении ответственного (пост-фильтрация). |
| **comment** | строка | Подстрока в комментарии (пост-фильтрация). |
| **summ** | число | Точное совпадение по сумме документа. |
| **approved** | булево | Фильтр по проведённости. Истина: `1`, `TRUE`, `YES`, `ДА`. |
| **full** | булево | При `true` — полный ответ: ТЧ Товары, полные списки связей, вложения в Base64. При `false` или отсутствии — базовый набор полей и сводки (Есть, Количество). |

По умолчанию (без параметров или с частичным набором) возвращаются все заказы, попадающие под переданные фильтры; при отсутствии периода (**from**/**to**) ограничения по дате нет.
