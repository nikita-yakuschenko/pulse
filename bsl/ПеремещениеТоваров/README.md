# Перемещение товаров (GetTransfers)

HTTP-сервис возвращает список документов **Перемещение товаров** (Документ.ПеремещениеТоваров) в виде единообразного JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetTransfers`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив перемещений: Номер, Дата, СкладОтправитель, СкладПолучатель, Ответственный, Комментарий; при full=true — Товары (ТЧ), иначе — поле Материалы в виде «N позиций») и `errors` (массив строк ошибок). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр. Форматы дат: **dd.MM.yyyy**, **dd-MM-yyyy**, **yyyy-MM-dd**.

| Параметр    | Назначение |
|-------------|------------|
| **code**    | Подстрока в номере документа (ПОДОБНО). При **full=true** параметр code обязателен. |
| **from**, **to** | Период по дате документа (оба обязательны для применения фильтра). |
| **date**    | Одна дата — фильтр по документу за указанный день (диапазон с начала до конца дня). |
| **responsible** | Подстрока в наименовании ответственного (ПОДОБНО). |
| **sender**  | Подстрока в наименовании склада-отправителя (ПОДОБНО). |
| **receiver**| Подстрока в наименовании склада-получателя (ПОДОБНО). |
| **full**    | При **true** в каждую запись добавляется табличная часть **Товары** (НомерСтроки, Номенклатура, ХарактеристикаНоменклатуры, ЕдиницаИзмерения, Количество). При **false** (по умолчанию) выводится сводка **Материалы** («N позиций»). |

В конфигураторе нужно задать шаблон URL, соответствующий имени функции **GetTransfers**, либо переименовать функцию под существующий шаблон.
