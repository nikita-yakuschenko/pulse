# Заказ покупателя (GetCustomerOrders)

HTTP-сервис возвращает список документов **Заказ покупателя** (Документ.ЗаказПокупателя) в виде JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetCustomerOrders`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив документов) и `errors` (массив строк ошибок). Даты в формате **dd.MM.yyyy**.

---

## Поля ответа

| Поле | Тип | Описание |
|------|-----|----------|
| Номер | Строка | Номер документа |
| Дата | Строка | Дата документа (dd.MM.yyyy) |
| Контрагент | Строка | Покупатель |
| Организация | Строка | Организация |
| Склад | Строка | Склад отгрузки |
| Ответственный | Строка | Ответственный менеджер |
| СуммаДокумента | Число | Сумма заказа |
| Проведен | Булево | Признак проведения |
| Комментарий | Строка | Комментарий |
| ДоговорКонтрагента | Строка | Договор |
| ВидОперации | Строка | Вид операции |
| ДатаОтгрузки | Строка | Плановая дата отгрузки |
| ДатаОплаты | Строка | Плановая дата оплаты |
| Товары | Объект/Массив | При `full=false`: `{Есть, Количество}`. При `full=true`: массив строк ТЧ |

---

## Параметры запроса

| Параметр | Назначение |
|----------|------------|
| **from**, **to** | Период по дате документа |
| **code** | Подстрока в номере документа (ПОДОБНО) |
| **contractor** | Подстрока в наименовании покупателя (пост-фильтр) |
| **org** | Подстрока в наименовании организации (пост-фильтр) |
| **warehouse** | Подстрока в наименовании склада (пост-фильтр) |
| **responsible** | Подстрока в наименовании ответственного (пост-фильтр) |
| **comment** | Подстрока в комментарии (пост-фильтр) |
| **approved** | `true/1/ДА` — только проведённые документы |
| **full** | `true/1/ДА` — полная табличная часть Товары |
