# Требование-накладная (GetDemands)

HTTP-сервис возвращает список документов **Требование-накладная** (Документ.ТребованиеНакладная) в виде единообразного JSON с полями **data** и **errors**.

**Функция в конфигураторе:** `GetDemands`. Имя шаблона URL должно совпадать с именем функции.

**Ответ:** JSON с полями `data` (массив документов: Номер, Дата, Склад, Ответственный, Комментарий, Материалы — сводка «N позиций» или при full=true массив строк ТЧ с НомерСтроки, Номенклатура, Количество, ЕдиницаИзмерения) и `errors` (массив строк ошибок). Даты в формате **dd.MM.yyyy**.

---

## Работа с параметрами

Параметры передаются через **ПараметрыЗапроса** или **ПараметрыURL**. Для параметра **full** учитывается разный регистр.

| Параметр        | Назначение |
|-----------------|------------|
| **code**        | Подстрока в номере документа (ПОДОБНО). |
| **from**, **to**| Период по дате документа (форматы даты, поддерживаемые 1С). |
| **warehouse**   | Подстрока в наименовании склада (ПОДОБНО). |
| **responsible** | Подстрока в наименовании ответственного (ПОДОБНО). |
| **full**        | При **true** в поле **Материалы** возвращается полная табличная часть (массив строк). При **false** (по умолчанию) — строка вида «N позиций». |

В конфигураторе нужно задать шаблон URL, соответствующий имени функции **GetDemands**, либо переименовать функцию под существующий шаблон.
