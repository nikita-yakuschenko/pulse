# Модуль.Пульс

Операционная система управления для производственной компании: производство панельно-каркасных (ПКД) и модульных домов (МД) + монтаж на объектах.

Покрывает полный операционный цикл — от закупок и расчёта потребности в материалах до управления складом, планирования монтажей и двусторонней интеграции с 1С:УПП.

---

## Разделы

### Закупки и снабжение

**Панель управления** — KPI-дашборд отдела закупок с 7 показателями эффективности: соблюдение бюджета, доля срочных закупок, срок закупки, доля брака, своевременность поставок, экономия, качество работы с поставщиками. Общий коэффициент KPI переводится в итоговую мотивацию. График динамики за последние 6 месяцев.

**Аналитика (MRP)** — расчёт потребности в материалах: пользователь выбирает спецификации из 1С, система суммирует потребность, вычитает текущие остатки и формирует список к закупке. Отчёты сохраняются в базе со статусом `draft` / `computed`, доступна детализация по каждому материалу и спецификации.

**Заказы поставщикам** — список заказов из 1С с фильтрами и постраничной навигацией. Скачивание файла заказа напрямую из 1С.

**Заявки на оплату (ЗРДС)** — реестр платёжных заявок из 1С. Скачивание файла платёжного поручения.

**Поступления** — приёмка товаров, привязка к заказам поставщикам, данные из 1С.

**Поставщики** — справочник контрагентов из 1С с возможностью добавить в избранное. Избранные поставщики выводятся в отдельной вкладке.

**Документация** — просмотр проектной документации (PDF и форматы Office) прямо в браузере. Спецификации номенклатуры из 1С с иерархическим деревом и поиском.

---

### Склад

**Номенклатура и остатки** — остатки по складам из 1С. Иерархическое дерево номенклатуры с возможностью отметить группы и материалы как избранные (выводятся наверху), скрывать ненужные группы, настраивать исключения из поиска. Настройки персонализированы по пользователю.

**Точки заказа** — настройка минимальных остатков для отдельных материалов или групп материалов (групповые точки заказа). Привязка к конкретным складам. Алерты при достижении пороговых значений.

**Движение материалов** — история перемещений материалов по складам из 1С с фильтрами по дате и складу.

---

### Производство

**График производства** — планирование производственных задач.

**Покрасочный цех** — управление очерёдностью и статусами покраски.

---

### Строительная часть

**Объекты** — реестр строительных объектов: адрес (стандартизация через DaData), договор, номер домокомплекта, тип (ПКД / МД), типовой проект, прораб, суммы по договору.

**График монтажа** — планирование выездных монтажей. Два режима в одном экране:

- *График монтажа* — объекты с утверждённой датой начала монтажа
- *Планирование работ* — объекты без даты, на согласовании

На каждый объект фиксируются ключевые этапы: запуск в производство, отгрузка, кровля, сдача. Отдельные поля для статуса производства, эскроу/ЕГРН, переноса суммы на следующий месяц. Автосборка статусной строки из заполненных дат.

Фильтры: месяц, адрес, тип здания, прораб.

## Интеграции

### 1С:УПП

REST API-слой с **95+ BSL HTTP-сервисами**, покрывающими документы, справочники и регистры. Клиент поддерживает два окружения (тест / продакшн), Basic Auth с кириллическими логинами, таймаут 120 с и один автоматический повтор при сетевой ошибке.

| Группа         | Объекты                                                                    |
| -------------- | -------------------------------------------------------------------------- |
| Закупки        | ЗаказПоставщику, ПоступлениеТоваровУслуг, СчётФактураПолученный, ЗРДС      |
| Продажи        | РеализацияТоваровУслуг, ЗаказПокупателя, СчётФактураВыданная               |
| Склад          | ПеремещениеТоваров, ИнвентаризацияТоваровНаСкладе, ПартииТоваровНаСкладах  |
| Производство   | ВыпускПродукции, ОтчётПроизводстваЗаСмену, ПередачаВПроизводство           |
| Зарплата       | НачислениеЗарплаты, ВыплатаЗарплаты, ПеречислениеЗарплатыВБанк             |
| ОС             | ПринятиеКУчётуОС, МодернизацияОС, СписаниеОС, НачислениеАмортизации        |
| Справочники    | Контрагенты, Сотрудники, Номенклатура, Склады, Валюты                      |
| Регистры       | РасчётыСПоставщиками, СебестоимостьТоваров, УчётРабочегоВремени            |

Ответы от 1С кешируются — Redis при наличии `REDIS_URL`, иначе in-memory с TTL 5 минут (до 200 записей). Это снижает нагрузку при повторных обращениях к медленным запросам.

### Telegram

Уведомления и оперативные оповещения. Поддерживается также как второй фактор аутентификации (OTP через бот).

### Bitrix24

Интеграция с CRM через webhook.

### DaData

Стандартизация и подсказки адресов при вводе в карточке объекта. Результат сохраняется в базе — повторных запросов к DaData не требуется.

## Аутентификация

Better Auth поверх Supabase Auth. Обязательное подтверждение email. Многофакторная аутентификация (MFA/2FA) — три метода на выбор:

- **TOTP** — приложение-аутентификатор (Google Authenticator, Authy и др.)
- **Email OTP** — одноразовый код на почту
- **Telegram OTP** — одноразовый код через бот

При входе пользователь сначала вводит пароль, затем проходит MFA. Без подтверждённого email и пройденного MFA (если включён) доступ в дашборд закрыт.

## Переменные окружения

| Переменная                      | Описание                                             |
| ------------------------------- | ---------------------------------------------------- |
| `DATABASE_URL`                  | PostgreSQL connection string (Supabase)              |
| `BETTER_AUTH_SECRET`            | Секрет подписи сессий (min 32 символа)               |
| `BETTER_AUTH_URL`               | Базовый URL приложения                               |
| `NEXT_PUBLIC_SUPABASE_URL`      | URL проекта Supabase                                 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Anon-ключ Supabase                                   |
| `TELEGRAM_BOT_TOKEN`            | Токен Telegram-бота (уведомления + MFA)              |
| `DADATA_API_KEY`                | API-ключ DaData (подсказки адресов)                  |
| `REDIS_URL`                     | Redis connection string (опционально, кеш 1С-данных) |

Настройки подключения к 1С и Bitrix24 хранятся в метаданных пользователя Supabase и задаются в разделе «Настройки → Интеграции» интерфейса.
